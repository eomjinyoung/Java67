실습 목표
1. 필터의 사용
- 서블릿 요청을 처리하기 전이나 후에 어떤 작업을 수행하고 싶을 때 사용하는 기술
- 사용법:
  1) javax.servlet.Filter 인터페이스를 구현한다.
  2) web.xml이나 애노테이션을 이용하여 배치 정보를 설정한다.
- Filter 인터페이스의 메서드
  init()      <-- 필터 생성 후 바로 호출됨. 필터에 필요한 자원을 준비시키는 작업을 두기에 적합하다.
  doFilter()  <-- 서블릿을 실행하기 전/후에 수행할 작업을 둔다.
  destroy()   <-- 웹 애플리케이션이 종료하거나 서버가 종료할 때 호출됨. 
                  필터를 위해 준비한 자원을 해제하는 코드를 두기에 적합하다.

- 필터의 사용 예)
  1) 서블릿 요청 처리 전
    => 요청 파라미터의 암호를 해제, 압축을 풀기, 특정 인코딩 데이터를 디코딩하기, 로깅하기 등
    => 권한 검사, 로그인 여부 검사, 임의 접근 차단 등
  2) 서블릿 요청 처리 후
    => 응답 데이터를 암호화하기, 압축하기, 인코딩하기, 로깅하기 등
  3) 공통
    => 서블릿 요청 전/후에 공통으로 수행할 작업이 있다면 필터를 사용하여 처리하라!

- 필터 기술
  1) 기능 확장이 쉽다.
    => 기능을 넣고 빼기 쉽다. 
    => 일반적으로 특정 기능을 추가하려면 해당 클래스에 코드를 추가해야 한다. 그 기능이 필요없을 때는
       각 클래스를 뒤져서 해당 코드를 제거해야 한다. => 유지보수가 번거롭다.
    => 그러나 필터를 사용하면, 기능을 객체화 했기 때문에(클래스로 정의) 넣고 빼기가 쉽다.
       => 하나의 기능에 하나의 클래스를 정의하기 때문에, 자잘한 클래스가 많이 생긴다.
       => 자잘한 클래스를 유지보수하기 번거로운 단점 보다는, 확장이 용이하다는 장점이 더 크다.
       
  2) 디자인 패턴의 Chain of Responsibility 기법과 같다.    

- 필터 실행 원리
서블릿 컨테이너 ---> 필터1.doFilter() ---> 필터2.doFilter() ---> 서블릿.service()
응답         <--- 리턴             <--- 리턴             <--- 리턴

- 필터 실행 원리(계층도)
서블릿 컨테이너 
            ---> 필터1.doFilter()
                 작업전..
                 ---------------> 필터2.doFilter()
                                  작업전...
                                  --------------------> 서블릿.service()
                                  <--------------------
                                  작업후...
                 <---------------
                 작업후
            <----
